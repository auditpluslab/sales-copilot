# E2Eテストの内容説明

## 📝 テストの目的

これらのテストは、**営業会議コパイロット**というWebアプリケーションの動作を自動で確認するものです。

---

## 🎯 アプリケーションの概要

**営業会議コパイロット**が何をするアプリか：

1. **会議中に音声を自動で文字起こし**（Whisper使用）
2. **AIが会話内容を分析**して、営業に役立つ情報を抽出
   - 顧客の課題（Pain Points）
   - 制約条件（予算、期間など）
   - ステークホルダーの姿勢
3. **リアルタイムで提案を表示**
   - 次に聞くべき質問
   - 提案カード

---

## 📊 テストの種類と内容

### 1️⃣ ホームページテスト（home.spec.ts - 9テスト）

**何をテストする？**:
- トップページが正しく表示されるか
- ボタンやリンクが動作するか

**具体的なテスト**:
```
✓ 「営業会議コパイロット」のタイトルが表示される
✓ 「新しいセッション」ボタンがある
✓ 「リアルタイム文字起こし」「インサイト抽出」「提案サポート」の説明カードがある
✓ ボタンをクリックするとセッション作成画面へ遷移する
✓ スマホ（iPhone）でも表示が崩れない
✓ タブレット（iPad）でも表示が崩れない
✓ PCでも正しく表示される
```

---

### 2️⃣ セッション作成テスト（session.spec.ts - 7テスト）

**何をテストする？**:
- 新しい会議セッションを作成する機能が動作するか

**具体的なテスト**:
```
✓ セッション作成フォームが表示される
✓ 必須項目（顧客名、会議タイトル）が空だと送信できない
✓ 顧客名、会社名、タイトルを入力できる
✓ 入力内容が長すぎるとバリデーションエラー
✓ 送信すると新しいセッションが作成される
✓ セッション一覧が表示される
✓ セッションがない場合のメッセージが表示される
```

---

### 3️⃣ セッションフローテスト（session-flow.spec.ts - 8テスト）

**何をテストする？**:
- セッション作成から会議ページ遷移までの一連の流れ

**具体的なテスト**:
```
✓ 新しいセッションを作成して会議ページへ遷移できる
  → ホーム → 入力 → 送信 → 会議ページ

✓ 必須項目未入力でバリデーションエラー

✓ セッション一覧が表示される

✓ セッションカードをクリックして会議ページへ

✓ フォーム入力がスムーズ（500ms以内）

✓ モバイルでフォームに入力できる

✓ APIエラー時にエラーメッセージが表示される

✓ ネットワークエラー時に適切に処理される
```

---

### 4️⃣ 会議ページテスト（meeting.spec.ts - 6テスト）

**何をテストする？**:
- 会議中の画面（会議ページ）の基本機能

**具体的なテスト**:
```
✓ 会議ページの構造（ヘッダー、タブ）が正しい
✓ タブ（文字起こし、インサイト、提案）を切り替えられる
✓ XSS攻撃を防いでいる
✓ SQLインジェクション攻撃を防いでいる
✓ ページが3秒以内に読み込まれる
✓ コンソールに重大なエラーが出ない
```

---

### 5️⃣ 会議フローテスト（meeting-flow.spec.ts - 16テスト）

**何をテストする？**:
- 会議ページでの完全な機能（インサイト、提案の表示）

**具体的なテスト**:
```
【会議開始・終了】
✓ 会議開始ボタンを押すと録音状態になる
✓ 会議終了ボタンで録音停止

【インサイト表示】
✓ インサイトタブで要約が表示される
✓ 顧客の課題（Pain Points）が表示される
  例: 「現行システムの遅さ（high）」
✓ 制約条件が表示される
  例: 「予算は年度内に確保必要」「導入はQ3を希望」
✓ ステークホルダーが表示される
  例: 「田中部長（IT部門）neutral」「佐藤課長（利用部門）champion」
✓ 顧客温度感（positive/neutral/negative）が表示される

【提案表示】
✓ 提案タブで「次に聞くべき質問」が表示される
  例: 「予算確保の具体的な時期について教えていただけますか？」
✓ 「提案カード」が表示される
  例: 「PoC実施提案」「段階的導入プラン」
✓ 信頼度（high/medium）が表示される

【文字起こし】
✓ 文字起こしのセグメントが表示される
  例: 「本日はお時間をいただきありがとうございます」
✓ 空の場合はメッセージが表示される

【UI/UX】
✓ タブ切り替えがスムーズ
✓ スクロールがスムーズ
✓ モバイルでも正しく表示される

【エラーハンドリング】
✓ インサイト読み込みエラー時の処理
✓ STT接続エラー時のリトライ
```

---

### 6️⃣ セキュリティテスト（security.spec.ts - 17テスト）

**何をテストする？**:
- アプリケーションのセキュリティ対策が機能しているか

**具体的なテスト**:
```
【入力バリデーション】
✓ SQLインジェクション攻撃を防ぐ
  → 「'; DROP TABLE sessions; --」を入力しても無効化される
✓ XSS攻撃を防ぐ
  → 「<script>alert("xss")</script>」を実行させようとしても無効化
✓ 過度に長い入力を拒否する

【APIセキュリティ】
✓ 認証なしのアクセスを制限
✓ CORSヘッダーが正しく設定されている
✓ セキュリティヘッダー（X-Frame-Optionsなど）が設定されている

【レート制限】
✓ 短時間に大量のリクエストを送ると制限される（429エラー）

【情報漏洩】
✓ エラーメッセージにデータベース情報が含まれない
✓ スタックトレースが公開されない

【セッション管理】
✓ CookieにHttpOnly属性が設定されている
✓ CookieにSameSite属性が設定されている

【機密情報】
✓ APIキーがHTMLソースに含まれない
✓ 環境変数が露呈しない
```

## 🎭 テストのイメージ（例）

### ホームページテストの実行イメージ

```
1. ブラウザで http://localhost:3000 を開く
2. 「営業会議コパイロット」の文字が見える ✓

4. /session/new に遷移する ✓
```

```
1. セッション作成画面を開く
2. 顧客名「テスト株式会社」と入力
3. 会議タイトル「初回ヒアリング」と入力
4. 「セッション開始」ボタンをクリック
5. 会議ページ /meeting/xxx へ遷移 ✓
```

### セキュリティテストの実行イメージ

```
1. 顧客名に「'; DROP TABLE sessions; --」を入力
2. 送信ボタンをクリック
3. データベースは削除されない ✓
4. バリデーションエラーが表示される ✓
```

## 📈 テストで保証すること

これらのテストを実行することで、以下を保証できます：

✅ **基本機能が動作する**
- ページが表示される
- ボタンがクリックできる
- フォームが送信できる

✅ **ユーザー体験が良好**
- スマホでも使える
- ページが速く読み込める
- エラーが分かりやすく表示される

✅ **セキュリティが守られている**
- 攻撃を受けても無効化される
- 情報漏洩がない
- 安全に入力できる

✅ **業務機能が正しく動作する**
- 音声認識が動く
- AI分析が機能する
- 提案が表示される
